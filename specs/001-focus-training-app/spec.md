## Clarifications

### Session 2026-03-29 (下午)

- **Q**: 玩法说明展示方式？ → **A**: 玩法说明固定在游戏页面上方以小字说明，不使用弹窗，避免打扰用户

### Session 2026-03-29

- **Q**: 主题系统默认行为？ → **A**: 默认跟随系统主题（system preference），暗色/亮色切换时确保文字可读性（WCAG 4.5:1 对比度标准）
- **Q**: 训练过程中是否提供操作提示？ → **A**: 所有训练模式不允许提示下一步该点击什么，用户需独立思考和操作完成关卡
- **Q**: 每个训练模式是否需要详细玩法介绍？ → **A**: 每个训练模式必须在开始前提供详细玩法说明，包括训练目标、操作方式、评分规则
- **Q**: 训练模式是否需要多种难度？ → **A**: 所有训练只保留困难模式，不允许用户设置难度
- **Q**: Stitch MCP 视觉设计范围？ → **A**: 优先核心页面 + 设计系统：首页 + 舒尔特表/字色干扰两个核心游戏 + 统计/设置页面，先生成设计系统确保视觉一致性
- **Q**: 最终测试使用的 MCP 工具？ → **A**: Chrome DevTools MCP - 用于端到端测试、性能验证、PWA 功能检查、响应式布局验证
- **Q**: 舒尔特表是否需要难度等级和多种网格大小？ → **A**: 不需要，舒尔特表固定使用 5x5 网格，保持当前交互和视觉设计

### Session 2026-03-28

- **Q**: 前端技术栈选择？ → **A**: 使用 React 19 + TypeScript 5.x + Vite + Tailwind CSS + shadcn/ui，Zustand 状态管理，IndexedDB 存储，Framer Motion 动画
- **Q**: 是否需要PWA支持？ → **A**: 完整PWA支持（Service Worker、离线缓存、manifest、可安装到主屏幕）
- **Q**: 动画风格偏好？ → **A**: 适度动画（页面过渡、按钮反馈、进度动画，保持专注，不干扰训练）
- **Q**: 本地存储方案选择？ → **A**: Dexie.js（IndexedDB封装，支持复杂查询、类型安全）
- **Q**: 测试方案选择？ → **A**: Vitest + React Testing Library（Vite原生支持，极速，现代方案）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 舒尔特表视觉训练 (Priority: P1)

用户在30秒内按升序或降序快速定位网格中的数字，训练视觉搜索效率与注意力集中度。

**Why this priority**: 舒尔特表是专注力训练的经典方法，规则简单易懂，适合作为核心训练模式吸引用户。

**Independent Test**: 可以独立测试通过完成舒尔特表训练并记录成绩来验证功能完整性。

**Acceptance Scenarios**:

1. **Given** 用户选择舒尔特表模式，**When** 点击开始训练，**Then** 系统显示数字网格并开始计时
2. **Given** 用户按顺序点击数字，**When** 点击正确数字，**Then** 数字标记为已完成并记录用时
3. **Given** 用户完成所有数字点击，**When** 计时结束，**Then** 系统显示成绩（用时、准确率）
4. **Given** 用户点击错误数字，**When** 发生错误点击，**Then** 系统给予视觉/声音反馈但不中断训练

---

### User Story 2 - 字色干扰抑制训练 (Priority: P1)

用户需要说出文字的颜色而非文字本身（如看到蓝色字写的"红色"要回答"蓝色"），通过抑制语义干扰激活前额叶皮层功能。

**Why this priority**: 字色干扰训练能有效提升注意力和多任务处理能力，是认知训练的经典范式。

**Independent Test**: 可以独立测试通过完成字色干扰题目并评分来验证功能完整性。

**Acceptance Scenarios**:

1. **Given** 用户选择字色干扰模式，**When** 开始训练，**Then** 系统显示带颜色的文字题目
2. **Given** 用户看到题目，**When** 选择/输入正确答案（文字颜色），**Then** 系统判断对错并记录
3. **Given** 用户完成一组题目，**When** 训练结束，**Then** 系统显示正确率、反应时间统计
4. **Given** 题目难度分级，**When** 用户连续正确，**Then** 系统提升难度（减少反应时间限制）

---

### User Story 3 - 序列工作记忆训练 (Priority: P2)

用户需要记忆物品品类及其出现的顺序，然后按顺序回忆，强化记忆编码与提取机制。

**Why this priority**: 序列记忆训练对学习能力提升有直接帮助，是重要的认知训练维度。

**Independent Test**: 可以独立测试通过完成序列记忆任务并评估记忆准确率来验证功能。

**Acceptance Scenarios**:

1. **Given** 用户选择序列记忆模式，**When** 开始训练，**Then** 系统依次展示物品及其顺序
2. **Given** 展示完成后，**When** 进入回忆阶段，**Then** 用户按顺序选择物品
3. **Given** 用户完成选择，**When** 提交答案，**Then** 系统显示记忆准确率（位置正确率、物品正确率）
4. **Given** 训练过程中，**When** 序列长度增加，**Then** 难度相应提升

---

### User Story 4 - 听觉选择性注意训练 (Priority: P2)

用户专注听取数字序列并记忆其出现顺序，锻炼听觉皮层与前额叶皮层功能，增强注意力的定向调控与抗干扰能力。

**Why this priority**: 听觉训练补充了视觉训练的不足，提供多感官训练体验。

**Independent Test**: 可以独立测试通过完成听觉数字记忆任务来验证功能。

**Acceptance Scenarios**:

1. **Given** 用户选择听觉训练模式，**When** 开始训练，**Then** 系统通过音频播放数字序列
2. **Given** 音频播放完毕，**When** 进入回答阶段，**Then** 用户按顺序输入/选择听到的数字
3. **Given** 训练环境有背景干扰，**When** 用户开启抗干扰训练，**Then** 系统添加背景噪音
4. **Given** 用户完成训练，**When** 查看成绩，**Then** 系统显示听觉记忆准确率

---

### User Story 5 - 双侧肢体镜像协调训练 (Priority: P3)

用户双手同步绘制镜像图形，通过运动皮层与小脑的协同训练，提高肢体协调性。

**Why this priority**: 镜像协调训练是独特的肢体协调训练，对思维流畅性有帮助。

**Independent Test**: 可以独立测试通过完成镜像绘图任务并评估对称性来验证功能。

**Acceptance Scenarios**:

1. **Given** 用户选择镜像协调模式，**When** 开始训练，**Then** 系统显示目标图形和双画布
2. **Given** 用户在左右画布上绘图，**When** 同时绘制，**Then** 系统实时评估图形对称性
3. **Given** 用户完成绘图，**When** 提交答案，**Then** 系统评分并显示对称度百分比
4. **Given** 训练过程中，**When** 难度提升，**Then** 目标图形复杂度增加

---

### User Story 6 - 规则导向分类逻辑训练 (Priority: P3)

用户依据属性对物品进行分类判断，优化逻辑思维与信息整合效率，提升决策速度。

**Why this priority**: 分类逻辑训练强化高级认知功能，是对专注力训练的有益补充。

**Independent Test**: 可以独立测试通过完成分类任务并评估反应速度和准确率来验证功能。

**Acceptance Scenarios**:

1. **Given** 用户选择分类训练模式，**When** 开始训练，**Then** 系统展示物品和分类规则
2. **Given** 用户看到物品，**When** 根据规则快速分类，**Then** 系统判断对错并记录反应时间
3. **Given** 规则发生变化，**When** 新规则生效，**Then** 系统提示规则切换
4. **Given** 训练结束，**When** 查看成绩，**Then** 系统显示分类准确率和平均反应时间

---

### User Story 7 - 情景联想记忆训练 (Priority: P3)

用户利用物品构建连贯的场景故事，强化记忆存储与关联能力，让信息记忆更牢固。

**Why this priority**: 联想记忆训练提供创造性记忆方法，与其他训练形成互补。

**Independent Test**: 可以独立测试通过完成情景构建任务并评估记忆效果来验证功能。

**Acceptance Scenarios**:

1. **Given** 用户选择联想记忆模式，**When** 开始训练，**Then** 系统展示一组随机物品
2. **Given** 用户看到物品，**When** 构建并输入情景故事，**Then** 系统保存用户的故事
3. **Given** 一段时间后，**When** 进行回忆测试，**Then** 系统评估用户对物品和顺序的记忆
4. **Given** 训练完成，**When** 查看成绩，**Then** 系统显示故事完整度和记忆准确率

### Edge Cases

- **边界条件**: 当用户未完成训练就退出时，系统应保存当前进度或询问是否保存
- **错误处理**: 当用户连续多次答错时，系统仅记录错误次数用于最终评分，不提供正确答案提示或操作引导
- **性能场景**: 当网络状况不佳时，训练功能应能离线正常运行
- **可访问性**: 色盲用户应能通过其他方式（如形状、图案）完成字色干扰训练
- **时间限制**: 训练应有合理的时长控制，防止用户过度疲劳

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须提供7种专注力训练模式：舒尔特表、字色干扰、序列记忆、听觉注意、镜像协调、分类逻辑、情景联想
- **FR-002**: 系统必须支持响应式设计，适配移动端和PC端屏幕
- **FR-003**: 所有训练模式固定为困难模式，不提供难度设置选项
- **FR-004**: 系统必须记录每次训练的成绩（用时、准确率、完成度）
- **FR-005**: 系统必须提供训练数据统计和进步趋势展示
- **FR-006**: 舒尔特表模式固定使用 5x5 网格，支持升序和降序两种模式，保持简洁的交互和视觉设计
- **FR-007**: 字色干扰模式必须提供颜色选项供用户选择，记录反应时间和正确率
- **FR-008**: 序列记忆模式必须支持调整序列长度（5-15个物品）
- **FR-009**: 听觉训练模式必须支持数字序列播放，可选添加背景干扰音
- **FR-010**: 镜像协调模式必须支持触摸/鼠标双画布同时绘图
- **FR-011**: 分类逻辑模式必须支持多种分类规则（颜色、形状、大小等）
- **FR-012**: 情景联想模式必须允许用户输入并保存自定义故事
- **FR-013**: 系统必须提供每日训练计划和提醒功能
- **FR-014**: 系统必须支持训练记录的本地存储
- **FR-015**: 系统默认主题必须跟随系统偏好（system preference），支持手动切换暗色/亮色模式，所有主题下文字对比度必须符合 WCAG AA 标准（4.5:1）
- **FR-016**: 所有训练模式不得提供"下一步点击什么"的操作提示，用户必须独立完成训练关卡，系统仅提供游戏规则说明和最终结果反馈
- **FR-017**: 每个训练模式必须在游戏页面上方固定展示玩法说明，使用小字简洁呈现训练目标、操作方式、评分规则，不使用弹窗

### Key Entities

- **训练记录**: 代表单次训练会话，包含训练模式、开始时间、用时、得分、难度级别
- **用户档案**: 代表用户信息，包含累计训练时长、各模式最高分、训练频率统计
- **训练配置**: 代表训练参数，包含难度设置、训练时长限制、提示开关
- **成绩统计**: 代表历史成绩汇总，包含日/周/月趋势、最佳成绩、平均表现

## Gameplay Details *(mandatory)*

### 1. 舒尔特表视觉训练 (Schulte Table)

**训练目标**：提升视觉搜索速度和注意力集中度

**玩法说明**：
1. 屏幕上显示 5×5 的随机数字网格（数字 1-25）
2. 训练开始后，用户需按升序（1→25）或降序（25→1）依次点击数字
3. 点击正确数字后，该数字标记为已完成
4. 点击错误数字时，系统记录错误但不中断训练，给予视觉/声音反馈
5. 完成所有数字点击后训练结束，显示用时和准确率

**评分规则**：
- 基础分：500分
- 时间奖励：完成越快得分越高
- 错误扣分：每次错误扣10分
- 难度系数：降序1.2x（固定困难模式）

**难度设置**：
- 固定为困难模式：降序模式

---

### 2. 字色干扰抑制训练 (Stroop Effect)

**训练目标**：提升抑制语义干扰的能力，激活前额叶皮层

**玩法说明**：
1. 屏幕中央显示一个带颜色的文字（如用蓝色显示的"红色"二字）
2. 用户需要选择文字的实际颜色，而非文字本身的含义
3. 从下方颜色选项中选择正确答案
4. 每轮训练包含15-20道题目
5. 系统记录每题的反应时间和正确率

**评分规则**：
- 基础分：每题50分
- 速度奖励：反应时间<1秒额外+20分
- 连续正确奖励：连续答对3题以上额外加分
- 总分 = 正确题数 × 基础分 + 速度奖励 + 连续奖励

**难度设置**：
- 固定为困难模式：反应时间限制1.5秒，20道题目

---

### 3. 序列工作记忆训练 (Sequence Memory)

**训练目标**：强化工作记忆和序列记忆能力

**玩法说明**：
1. 系统依次展示一系列物品（emoji图标），每个物品显示1秒
2. 物品展示完毕后进入回忆阶段
3. 用户需要按照刚才展示的顺序依次点击物品
4. 选择的物品会按点击顺序排列在上方
5. 完成选择后提交答案

**评分规则**：
- 位置准确率：物品在正确位置得满分
- 物品准确率：物品正确但位置错误得半分
- 基础分：500分
- 难度系数：1.5x（固定困难模式，10个物品）

**难度设置**：
- 固定为困难模式：10个物品序列

---

### 4. 听觉选择性注意训练 (Auditory Attention)

**训练目标**：提升听觉注意力和抗干扰能力

**玩法说明**：
1. 系统通过语音播放一串数字序列（如：3-7-1-9）
2. 用户仔细聆听并记忆数字顺序
3. 播放完毕后，用户按顺序输入听到的数字
4. 可选择开启背景白噪音增加难度
5. 提供2次重播机会

**评分规则**：
- 基础分：300分
- 正确率奖励：每正确一个数字+100分
- 干扰 bonus：开启白噪音额外x1.2倍
- 难度系数：1.6x（固定困难模式，7位数字）

**难度设置**：
- 固定为困难模式：7位数字

---

### 5. 双侧肢体镜像协调训练 (Mirror Coordination)

**训练目标**：提升双侧肢体协调性和空间感知能力

**玩法说明**：
1. 屏幕分为左右两个画布区域
2. 左侧显示目标形状（线条、圆形、方形、三角形）
3. 用户在左侧画布上沿虚线描绘目标形状
4. 右侧画布实时显示用户的镜像绘制轨迹
5. 绘制时间限制30秒
6. 完成后提交，系统评估对称性和完整性

**评分规则**：
- 对称性分数（60%权重）：左右轨迹的匹配度
- 完整性分数（40%权重）：对目标形状的覆盖程度
- 总分 = 对称性得分 × 0.6 + 完整性得分 × 0.4

**难度设置**：
- 固定为困难模式：三角形

---

### 6. 规则导向分类逻辑训练 (Categorization Logic)

**训练目标**：提升认知灵活性和快速决策能力

**玩法说明**：
1. 屏幕中央展示一个物品（具有颜色、形状、大小三个属性）
2. 上方显示当前分类规则（按颜色/形状/大小）
3. 用户根据当前规则判断该物品属于哪一类
4. 点击下方三个分类按钮之一（颜色/形状/大小）
5. 规则会随机切换，用户需要快速适应新规则
6. 每轮训练15个物品

**评分规则**：
- 基础分：每题40分
- 速度奖励：反应时间<1.5秒额外+20分
- 规则适应奖励：规则切换后首次答对额外+30分
- 总分 = 正确数 × 基础分 + 速度奖励 + 适应奖励

**难度设置**：
- 固定为困难模式：每3-5个物品随机切换规则

---

### 7. 情景联想记忆训练 (Scenario Association)

**训练目标**：提升联想记忆和创造性记忆能力

**玩法说明**：
1. 系统展示一个场景（如厨房、办公室、公园）和该场景中的5-7个物品
2. 用户有15秒时间记忆这些物品
3. 记忆阶段结束后，用户选择记住的物品
4. 然后用户用选中的物品编一个连贯的场景故事
5. 系统评估回忆准确率和故事完整度

**评分规则**：
- 回忆准确率：正确回忆的物品占总物品的比例
- 故事完整度：故事包含的回忆物品数量
- 基础分：300分
- 回忆 bonus：准确率 × 4分
- 故事 bonus：完整度 × 3分

**难度设置**：
- 固定为困难模式：7个物品，10秒记忆时间

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可以在2分钟内开始任意训练模式，无需复杂设置
- **SC-002**: 页面加载时间不超过3秒（在标准网络环境下）
- **SC-003**: 移动端和PC端的界面交互均流畅，无明显的布局问题

## Assumptions

### Visual Design Strategy (Stitch MCP)

- **设计范围**: 使用 Stitch MCP 生成核心页面视觉设计
  - Phase 1: 设计系统（颜色、字体、圆角、间距规范）
  - Phase 2: 首页（游戏选择、统计概览）
  - Phase 3: 舒尔特表游戏界面
  - Phase 4: 字色干扰游戏界面
  - Phase 5: 统计页面 + 设置页面
- **扩展策略**: 其他5个游戏基于已建立的设计系统和核心页面模式自行实现，确保一致性
- **平台适配**: Stitch 设计以移动端优先（375px），同时确保在PC端（1280px+）的可用性
- **设计风格**: 简洁专业、专注训练氛围、适度动画反馈、高对比度确保可读性

### Technical Assumptions

- 目标用户具有基本的智能设备操作能力
- 用户拥有稳定的网络连接用于初始加载，但训练过程可离线进行
- 用户使用现代浏览器（Chrome、Safari、Edge、Firefox最新2个版本）
- 移动端用户主要使用手机进行训练，PC端用户使用鼠标/键盘
- 用户训练的主要目的是提升专注力而非娱乐，因此界面应简洁专业
- 训练数据主要存储在本地，无需云端账户系统（第一阶段）
- 音频训练模式需要设备支持音频播放
- 镜像协调训练在移动端需要多点触控支持
- 用户每天训练时间建议控制在20-30分钟内，系统应提供时长建议
- UI设计使用Stitch MCP生成，确保视觉风格统一且专业
- **技术栈**: React 19 + TypeScript 5.x + Vite + Tailwind CSS + shadcn/ui
- **状态管理**: Zustand（轻量状态管理）
- **数据存储**: IndexedDB（via Dexie.js）用于离线支持
- **动画**: Framer Motion（流畅交互体验）
- **音频**: Howler.js（跨浏览器音频支持）

### Testing Strategy (Chrome DevTools MCP)

- **单元测试**: Vitest + React Testing Library 测试组件和 hooks
- **E2E 测试**: Chrome DevTools MCP 进行端到端测试
  - 游戏流程自动化（舒尔特表点击序列、计分验证）
  - PWA 功能验证（离线模式、Service Worker、manifest）
  - 性能指标检查（LCP、FID、CLS、FPS）
  - 响应式布局验证（移动端 375px / 桌面端 1280px）
  - 可访问性审计（颜色对比度、键盘导航）
- **测试覆盖目标**: 核心功能 80%+ 覆盖率
